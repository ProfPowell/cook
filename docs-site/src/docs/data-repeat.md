# Data Repeat

The `data-repeat` attribute generates repeated HTML from arrays in your data. Write the template once and Cook stamps it out for each item at build time.

## Basic usage

Add `data-repeat` to any element. The value specifies which array to iterate and a name for the current item:

<code-block language="javascript" label="config/data.js">export default {
  features: [
    { title: 'Fast Builds', desc: 'Sub-second builds for most sites' },
    { title: 'Zero JS', desc: 'No client-side framework required' },
    { title: 'Markdown', desc: 'Write content in Markdown with frontmatter' },
  ],
};</code-block>

<code-block language="html" label="src/index.html">&lt;ul&gt;
  &lt;li data-repeat="features as feature"&gt;
    &lt;strong&gt;&#36;{feature.title}&lt;/strong&gt; &amp;mdash; &#36;{feature.desc}
  &lt;/li&gt;
&lt;/ul&gt;</code-block>

Cook outputs one `<li>` for each item, with template strings resolved:

<browser-window url="Build Output">&lt;ul&gt;
  &lt;li&gt;&lt;strong&gt;Fast Builds&lt;/strong&gt; &amp;mdash; Sub-second builds for most sites&lt;/li&gt;
  &lt;li&gt;&lt;strong&gt;Zero JS&lt;/strong&gt; &amp;mdash; No client-side framework required&lt;/li&gt;
  &lt;li&gt;&lt;strong&gt;Markdown&lt;/strong&gt; &amp;mdash; Write content in Markdown with frontmatter&lt;/li&gt;
&lt;/ul&gt;</browser-window>

## Nested data access

Use dot notation to reference arrays inside nested objects:

<code-block language="html" label="src/team.html">&lt;div data-repeat="company.team as member"&gt;
  &lt;h3&gt;&#36;{member.name}&lt;/h3&gt;
  &lt;p&gt;&#36;{member.role}&lt;/p&gt;
&lt;/div&gt;</code-block>

## Limit and offset

Control how many items to render and where to start with pipe modifiers:

<code-block language="html" label="Show only the first 3 items">&lt;article data-repeat="posts | limit:3 as post"&gt;
  &lt;h2&gt;&#36;{post.title}&lt;/h2&gt;
&lt;/article&gt;</code-block>

<code-block language="html" label="Skip 2, then show 5">&lt;article data-repeat="posts | offset:2 | limit:5 as post"&gt;
  &lt;h2&gt;&#36;{post.title}&lt;/h2&gt;
&lt;/article&gt;</code-block>

Offset is applied first, then limit. So `offset:2 | limit:5` skips the first 2 items and takes the next 5.

## Index variable

Access the current iteration index by adding a second variable name after a comma:

<code-block language="html" label="src/index.html">&lt;ol&gt;
  &lt;li data-repeat="steps as step, i"&gt;
    Step &#36;{i}: &#36;{step.text}
  &lt;/li&gt;
&lt;/ol&gt;</code-block>

The index is zero-based.

## Template strings in attributes

Template variables work in attributes on the repeated element itself, not just in its children:

<code-block language="html" label="src/blog.html">&lt;a data-repeat="posts as post" href="&#36;{post.url}" class="post-link"&gt;
  &lt;h2&gt;&#36;{post.title}&lt;/h2&gt;
  &lt;time&gt;&#36;{post.date}&lt;/time&gt;
&lt;/a&gt;</code-block>

## Using with collections

Markdown collections built during the build process are available through the `collections` namespace. If you have Markdown files in `src/blog/`, Cook auto-generates a `collections.blog` array:

<code-block language="html" label="src/blog.html">&lt;article data-repeat="collections.blog as post"&gt;
  &lt;h2&gt;&lt;a href="&#36;{post.url}"&gt;&#36;{post.title}&lt;/a&gt;&lt;/h2&gt;
  &lt;p&gt;&#36;{post.description}&lt;/p&gt;
&lt;/article&gt;</code-block>

Collection items have properties from their front matter (`title`, `description`, `date`, etc.) plus a `url` field generated by Cook.

## Using with components

Repeat runs **before** component expansion in the build pipeline. This means you can repeat a custom element and each copy gets expanded individually:

<code-block language="html" label="src/products.html">&lt;product-card
  data-repeat="products as product"
  name="&#36;{product.name}"
  description="&#36;{product.desc}"
  href="&#36;{product.url}"&gt;
&lt;/product-card&gt;</code-block>

Each repeated `<product-card>` is expanded into its full component template with the correct attribute values.

## Expression syntax

The full expression format is:

```
dataPath | offset:N | limit:N as itemName, indexName
```

- `dataPath` -- dot-notation path to an array (e.g., `features`, `collections.blog`)
- `offset:N` -- optional, skip the first N items
- `limit:N` -- optional, take at most N items
- `itemName` -- required, variable name for the current item
- `indexName` -- optional, variable name for the zero-based index

The `data-repeat` attribute is removed from the output by default.
